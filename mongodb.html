Mongo DB

What is MongoDB, and how is it different from traditional relational databases?

MongoDB is a NoSQL database that stores data as flexible JSON-like documents. Unlike traditional relational databases, MongoDB does not require a predefined schema, scales horizontally, and uses a query language suitable for JSON-like structures.

.Can you explain MongoDB document and a collection?

  • MongoDB Document: A MongoDB document is a data record stored in a collection. It's a JSON-like structure that can contain various field-value pairs.
  • MongoDB Collection: A MongoDB collection is a grouping of MongoDB documents. It's analogous to a table in relational databases but does not enforce a schema across documents.

    • How do you create, read, update, and delete documents in MongoDB?

  • Flexible Structures: MongoDB allows varying document structures without a fixed schema.
  • Embedded Data: Use embedded documents and arrays to store related data together.
  • Normalization: Decide between embedding data for simplicity or referencing for scalability.
  • Schema Validation: Optionally enforce document structure rules to maintain data integrity.
  • Adaptability: Design to accommodate future data changes and evolutions.
